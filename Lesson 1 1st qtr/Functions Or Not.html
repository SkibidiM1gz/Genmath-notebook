Functions

Now, what is a function?

A function is a type of relation or a rule of correspondence between two nonempty sets such that, to each element of the first set called domain, there corresponds one and only one element of the second set called range.

Remarks:

A function is said to be a ‚Äúwell-behaved‚Äù relation. Why? This is because given a starting point, we know exactly where to go. Given a value, say x, we get only and exactly one value of y.

This means that while all functions are relations (since they pair information), not all relations are functions. In fact, functions are a sub-classification of relations.

DESCRIPTIVE RULE: FUNCTION OR NOT?

  1. the rule which assigns to each person his aunt/s
    This is NOT A FUNCTION since a person can have more than one aunts.
  2. the rule which assigns to each car to its plate number
    This is a FUNCTION since a car can only have one plate number
  3. the rule which assigns each student to his/her moderator
    This is a FUNCTION since many students can be assigned to a one moderator.

SET OF ORDERED PAIRS: FUNCTION OR NOT?

{ (SUPERMAN, DC), (BATMAN, DC), (THE FLASH, DC), (SPIDERMAN, MARVEL), (CAPTAIN AMERICA, MARVEL), (WOLVERINE, MARVEL) }

This set of ordered pairs is a FUNCTION.

For a set of ordered pairs to be a FUNCTION, there should be NO REPETITION OF THE X-COORDINATE.

GRAPH: FUNCTION OR NOT?

If we are given a graph, how do we know if it‚Äôs a function or not?

  • Vertical Line Test: A graph represents a function if any vertical line drawn through it intersects the graph at most once.

It is used to visually determine if a graph represents a function. Just slide any vertical line (think of your pencil)
across the graph to see if any two or more points of the graph lie on the same vertical line.

If there are no two or more points on the same vertical line, then the relation is a FUNCTION.

If there are two or more points on the same vertical line then the relation is NOT A FUNCTION.
